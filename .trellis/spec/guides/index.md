# 共享思维指南

这些指南适用于跨业务 app 或同时修改 Django 与 React 的任务。先读目标 app 的规范，再按任务是否跨层、是否重复实现选择指南。

## 可用指南

| 文档 | 适用场景 |
|---|---|
| [代码复用思维](./code-reuse-thinking-guide.md) | 新增 helper、常量、状态动作或发现重复逻辑 |
| [跨层数据流思维](./cross-layer-thinking-guide.md) | API、前端、数据库、缓存、WebSocket 或 Celery 同时变化 |

## 快速触发条件

- 新建工具函数、常量或配置前：阅读代码复用指南并用 `rg` 搜索已有定义。
- 需求涉及 API 与页面、订单状态与异步任务、缓存与持久化：阅读跨层指南。
- 修改响应字段或事件载荷：追踪所有生产者和消费者，不能只改当前文件。
- 审查意见涉及安全、权限或“缺少校验”时：先确认真实数据来源和信任边界，再决定是否修改。

## 通用原则

1. 规范记录当前代码库已经采用的模式，不把未落地的重构愿望当作规则。
2. 重要结论必须能指向真实源文件或测试；发现新模式后同步更新对应规范。
3. 代码、注释、测试名称和本目录文档使用中文，技术标识符保持项目已有英文命名。
