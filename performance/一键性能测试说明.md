# 一键性能测试说明

`run_performance_test.ps1` 会自动完成以下工作：

1. 执行数据库迁移。
2. 生成独立的性能测试用户、商品、订单与私信数据。
3. 为每个压测用户生成独立短期 JWT，避免登录接口和订单创建接口的单账号限流影响结果。
4. 运行 1 分钟缓存预热。
5. 运行核心读接口压测。
6. 运行创建订单并发写入压测。
7. 在“测试结果”目录生成中文 Markdown 测试报告，并将 HTML、CSV、预热文件和含 JWT 的配置隔离到“中间文件”目录。

## 运行前提

- 后端已运行在 `http://127.0.0.1:8000`。
- Redis、数据库均为本地开发或专用测试实例。
- 不能连接生产数据库。
- 项目虚拟环境已安装 Locust。

## 最简单的运行方式

在项目根目录打开 PowerShell 并执行：

```powershell
powershell -ExecutionPolicy Bypass -File performance\run_performance_test.ps1
```

默认使用中档数据、20 个读接口虚拟用户、每秒启动 5 个用户、读接口持续 5 分钟。整个过程约 7 分钟：1 分钟预热、5 分钟读接口测试和约 1 分钟订单创建测试。

大档数据包含 2,000 用户、50,000 商品、20,000 历史订单、100,000 私信和 1,500 个订单创建商品。大档默认将订单创建测试延长为 5 分钟：

```powershell
powershell -ExecutionPolicy Bypass -File performance\run_performance_test.ps1 -Profile large
```

需要调整订单创建测试时长时，可传入 `-WriteDuration`，例如 `-WriteDuration 10m`。

首次确认流程时，可改用小档数据和较短时长：

```powershell
powershell -ExecutionPolicy Bypass -File performance\run_performance_test.ps1 `
  -Profile small -ReadUsers 10 -SpawnRate 1 -ReadDuration 1m
```

## 测试结果在哪里

每次运行都会生成一组同名的独立目录：

```text
performance/测试结果/性能测试_日期时间/性能测试报告.md
performance/中间文件/性能测试_日期时间/
```

最先查看 `性能测试报告.md`，它以中文汇总各接口请求数、失败数、平均响应时间、P95、P99 与 RPS。若需要进一步分析趋势和响应分布，再打开“中间文件”目录中的 `核心读接口.html` 或 `创建订单.html`。

“中间文件”目录中的 `测试配置.json` 含本次生成账号的短期 JWT，仅用于 Locust 运行，不能提交到 Git 或发送给他人。

## 当前覆盖范围

- 商品列表、商品搜索、匿名商品详情。
- 分类列表缓存。
- 卖家公开主页及评分摘要。
- 买家订单列表与卖家订单列表，均使用有真实历史订单的独立账号。
- 私信会话列表和最新消息窗口缓存。
- 创建订单：独立账号、独立虚拟商品、每人仅一笔订单。

订单支付、发货、确认收货、评分、商品发布、图片上传和 WebSocket 通知不在此脚本范围内。这些操作会改变状态或需要专项数据准备，应作为单独的业务流程压测实施。
